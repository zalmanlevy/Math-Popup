Overview

Math Popup is a small always-available notepad. Each line is evaluated as a math expression and the result appears in the right gutter. You can use it like a back-of-the-envelope calculator that keeps a running history, or switch it into plain Text mode and use it as a tray-resident scratchpad.

The window lives in the system tray. Click the tray icon to toggle it, or press Ctrl+Alt+M from anywhere.

Window controls

The header from left to right:

  • Drag handle — click and drag anywhere in the header to move the window.
  • Math / Text — switch the editor between math evaluation and plain text.
  • 📌 Pin — keep the window above other apps. Highlights when active.
  • ƒ Variables — hover to see every variable currently defined in the note and its current value. Details in the Variables popup section below.
  • ? — opens this help page.
  • ⚙ — opens Settings.
  • × — hides the window. The app stays in the tray.

The window is resizable from any edge. Position and size are remembered between sessions.

Math & text modes

Math mode shows two gutters: line numbers on the left (L1, L2, …) and the evaluated result on the right. Syntax highlighting colors numbers, operators, variables, line refs, parentheses, and percent signs.

Text mode hides the result column and turns off math evaluation. Markdown styling (headers, bullets, bold, italic) still renders so you can take notes comfortably.

Numbers & formatting

  • Type any expression: (12 + 5) * 3 → 51.
  • Bare decimals get a leading zero automatically: .5 → 0.5.
  • Numbers with 4+ digits get commas inserted on space, operator, or Enter: 1000 → 1,000. Toggleable in Settings.
  • Commas in input are ignored when evaluating, so 1,000 + 1,000 works.
  • Results are formatted with commas and trimmed to your decimal cap.

Currency symbols

Money symbols are recognized inside expressions and ignored when the line is evaluated, so you can write what feels natural and still get a result:

  • $50 + $20 → 70
  • £1,000 * 1.05 → 1,050
  • €500 - 5% → 475

Recognized: $ € £ ¥ ₹ ₽ ¢ ₩ ₪ ₫ ₴ ₸ ₺ ฿. They render with a muted color in the editor so you can see they're decorative.

Directives

Directives are special lines starting with / that change how the lines below them are rendered. They produce no result themselves and are styled with a colored pill so they're easy to spot.

  • /no_dec_limit — overrides your Max decimal places setting for everything below this line. Results show their natural number of decimals (trailing zeros are stripped) capped at 6, BA II Plus-style. Use it when you need more precision for one section without opening Settings.
  • /clear — wipes the entire note (useful when you've finished a calculation and want a blank slate).

Example with Max decimal places set to 2:

10 / 3        → 3.33
/no_dec_limit
10 / 3        → 3.333333
1 / 2         → 0.5
6             → 6

The /no_dec_limit directive must be on its own line and applies until the end of the note — there's no reverse directive yet.

Slash command menu

Type / at the start of any line and a popup appears with all available slash commands. Continue typing to filter, use ↑/↓ to move, and Enter or click to confirm. Esc dismisses it.

The menu also flips above the caret when there isn't enough room below, and shows up to 5 items at a time (longer lists scroll).

Available commands today: /no_dec_limit and /clear — see the Directives section above.

Suffixes (k, m, b, t)

Type a number followed by a suffix, then press space — the editor expands it to the full number:

  • 5k → 5,000
  • 1.5m → 1,500,000
  • 2b → 2,000,000,000
  • 3t → 3,000,000,000,000

You can add or edit your own suffixes in Settings → Custom suffixes. Each suffix has a symbol, a multiplier, and a case-sensitivity flag.

If you turn off "Expand suffixes in editor", the typed text stays as 5k but the evaluator still computes the right value.

Smart Tab keeps the suffix attached. When you press Tab with the caret inside a number that has a trailing suffix (e.g. 10000k), the caret jumps past the entire token — the k is treated as part of the number, not separated by a space.

Variables

Define a variable on its own line, then reference it later:

price = 12.50
qty   = 7
price * qty

Variable names use letters, digits, and underscores, and must start with a letter or underscore. Reserved names like L1, pi, e, true, and false cannot be redefined — trying to assign to one shows Reserved in the result column with a hover tooltip.

Variables popup (ƒ)

The ƒ button in the title bar opens a hover popup listing every variable currently defined in the note, along with its latest value and the line it was assigned on. Hover the button to open it; move the cursor onto the popup to keep it open.

Each row shows:

  • The variable name (with a small L<n> badge for the line where it was last assigned).
  • The current evaluated value, formatted the same way as the result gutter.

If a variable is reassigned later in the note, only the most recent value is shown — matching the way the evaluator updates the variable line by line as it works through your expressions.

Variables that errored or weren't fully defined yet are skipped, and the popup says "No variables defined yet." when the note doesn't have any.

Line references

Refer to the result of any line with L followed by the line number. Case doesn't matter — L1 and l1 both work:

1,000 + 250
L1 * 0.07

Line-ref popup: typing L on its own (not as part of a longer word) opens a picker showing every line that has a numeric result, e.g. L1: 1,250. Use the arrow keys to navigate, Enter or click to insert the reference. The popup follows the same flip-above / 5-item rules as the slash menu.

Forward references work too. A line on row 2 can reference L3 or any later line — the evaluator runs in passes until results stabilize, so forward refs resolve as long as there's no circular dependency.

Auto-shifting: when you insert or delete a whole line, every L<n> reference (including ranges like L1:L4) on surrounding lines is rewritten so they keep pointing at the same target.

Line ranges (L1:L4)

Aggregate functions accept a colon-separated range of line refs that expands to "every line from A through B". Empty / blank / erroring lines inside the range are skipped, matching Excel's behavior:

10
20
30
SUM(L1:L3)        → 60
AVERAGE(L1:L3)    → 20

x as multiplication

The letter x works as a multiplication operator anywhere between numbers, parens, or other identifiers:

  • 5x10 → 50
  • (L1 + L2) x 3
  • $25 x 4 → 100

Because x is reserved as an operator, you can't define a variable called x. If you try, the result column shows N/A with a hover tooltip explaining why.

Percent & basis points

% after a number means "of one": 50 * 20% → 10. With a leading + or - it becomes "relative to the left side":

  • 200 + 10% → 220 (10% of 200, added)
  • 200 - 5% → 190

bps (or bp) is basis points — one hundredth of a percent. Same relative behavior:

  • 1,000,000 + 25bps → 1,002,500
  • 50bps → 0.005

IF, TRUE, and FALSE

IF(condition, value_if_true, value_if_false) works the way you'd expect from Excel, with a few quality-of-life conveniences:

  • = means equality inside expressions — IF(L1=L2, …) compares L1 and L2. The line-level name = expr form is unaffected (that's still an assignment).
  • Bare TRUE and FALSE (uppercase) act as the string literals "TRUE" and "FALSE", so you don't have to wrap them in quotes: IF(L1=L2, TRUE, FALSE).
  • Defaults: if you omit the second and third arguments, IF returns "TRUE" when the condition is truthy and "FALSE" otherwise: IF(L1=L2) → TRUE or FALSE.
  • Other strings need quotes. If you write IF(L1=L2, YES, NO) the result column shows red err with a hover tooltip "Add Quotations" — wrap the values: IF(L1=L2, "YES", "NO").

Excel = prefix

If you have Excel muscle memory, you can prefix any formula with = and it's stripped before evaluation:

  • =SUM(L1:L4) — same as SUM(L1:L4)
  • =L1 * 1.05 — same as L1 * 1.05
  • =TODAY() — same as TODAY()

Error indicators

The result column shows different markers for different problems. Hover over any of them to see a styled tooltip explaining the issue:

  • N/A — you tried to assign to x, which is reserved as a multiplication operator.
  • Excel Formula (red) — you tried to assign to a name that's reserved as an Excel-style function (sum, round, etc.). Tooltip lists every reserved name.
  • Reserved — you tried to assign to a reserved name like a line ref (L1 = 5), a math constant (pi = 5), or a literal (true = 5). These names can't be used as variables.
  • err (red) — usually means a bare word the evaluator doesn't recognize. Tooltip says "Add Quotations" when it looks like an unquoted text value (the only words that work without quotes are TRUE and FALSE).
  • blank with red underline in the editor — parse-in-progress: you're mid-typing an expression that isn't finished yet (e.g. SUM(). The result column stays empty so it doesn't flash error noise while you type.

Selection footer (Sum & Avg)

Highlight several lines in the editor and the status bar at the bottom shows the sum and average of the numeric results in your selection — handy for quick totals without writing a formula:

10
20
30
             ← select rows 1–3
 
                Sum: 60   •   Avg: 20

Lines without a numeric result are skipped. Single-line selections fall back to the normal "Ready" / "N errors" status.

Functions & constants

Powered by mathjs, so you have access to a large standard library. Common built-ins recognized by the syntax highlighter:

  • Constants: pi, e, tau, phi
  • Trig: sin, cos, tan, asin, acos, atan, atan2
  • Logs & exp: log, log2, log10, ln, exp
  • Roots & rounding: sqrt, abs, round, floor, ceil
  • Aggregates: min, max, sum, mean, median, mod

Examples: sqrt(2), sin(pi / 4), round(123.456, 1).

Markdown

Light markdown styling renders inline:

  • # Heading, ## Heading, ### Heading — header lines are not evaluated.
  • - item or * item — bulleted line. The content after the bullet is still evaluated as math (in math mode).
  • **bold** and *italic* — inline emphasis.

Tray & startup

  • Click the tray icon to toggle the popup. Double-click works too.
  • Right-click the tray icon for a menu: Open, Settings, Quit.
  • Closing the window only hides it — the app stays in the tray. Use the tray menu's "Quit" to exit.
  • "Launch when Windows starts" puts the app in your login items. It starts hidden, so only the tray icon appears until you click it.

Settings

  • Theme — System (follows your OS), Light, or Dark. System mode updates automatically when you change your OS color scheme.
  • Launch when Windows starts — auto-start hidden in the tray.
  • Auto-format numbers with commas — adds thousands separators as you type.
  • Expand suffixes in editor — turns 5m into 5,000,000 on space. Off keeps the typed form.
  • Max decimal places — caps how many decimals are shown in results. Integers omit the decimal point entirely.
  • Custom suffixes — add your own multipliers (e.g. x = 1000, kg = 1, etc.). Each row has a symbol, multiplier, and case-sensitive toggle.

Storage & privacy

Everything is stored locally on your machine. Your note text, window position, and settings are saved to a settings.json file in the app's user-data folder. Nothing is sent over the network.

Your note auto-saves a few hundred milliseconds after you stop typing.
